# Geo-Schedule Optimizer — Report Template

## Changelog: `context/analysis/geo-schedule-changelog.md`

This file is managed by `mutate.js` automatically. Each live-mode run appends an entry.

```markdown
## {YYYY-MM-DD} — Geo-Schedule Optimizer

**Mode:** live | **Account:** {customer_id}
**Result:** {applied}/{total} applied

| Campaign | Action | Dimension | Target | Before | After | Status |
|----------|--------|-----------|--------|--------|-------|--------|
| {name} | update | location | {location} | +0% | +25% | OK |
| {name} | create | location_exclusion | {location} | — | Excluded | OK |
| {name} | create | ad_schedule | Mon 01:00-05:00 | — | -100% | OK |
| {name} | update | device | TABLET | +0% | -45% | FAILED |
```

## Operations File: `created/geo-schedule-ops/operations-{YYYY-MM-DD}.json`

Preserved after each run (both dry-run and live). Contains:
- Full operations array with meta fields
- Description and source audit report reference
- Can be re-run later if needed
