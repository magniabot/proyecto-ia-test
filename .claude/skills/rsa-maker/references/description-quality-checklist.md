# Description Quality Checklist — Reference

Run after composing all descriptions. Validates constraints, quality, and angle coverage.

## Constraints

- [ ] Each description <= 90 characters
- [ ] Most descriptions 75-90 characters (space utilized)

## Standalone Validity

- [ ] Each description makes sense on its own
- [ ] No awkward fragments or broken phrases
- [ ] Grammar and punctuation are clean

## Non-Duplication

- [ ] No description simply restates a headline in different words
- [ ] At least one description adds proof, specificity, or terms not in headlines
- [ ] Descriptions do not duplicate sitelinks/callouts repeatedly

## Quality

- [ ] Searcher-focused (benefits/outcomes), not "we are..."
- [ ] No vague claims ("best", "quality service", "great results") without specifics
- [ ] No unverifiable superlatives or policy-risk phrases
- [ ] At least one description includes evidence (numbers, ratings, credentials)

## Angle Coverage

- [ ] At least one description includes risk reduction (trial, refund, no obligation)
- [ ] At least one description includes a clear CTA or next step
- [ ] At least 2 distinct angles across descriptions (not all same theme)
- [ ] A mix of problem+solution and aspirational/value messaging exists

## Keyword Bolding (Optional Optimization)

- [ ] At least one description includes the core keyword phrase naturally
- [ ] Keyword inclusion reads grammatically (not stuffed)
