# Offer Angle Quality Checklist — Reference

Used by the offer-auditor when evaluating angle extraction quality (if `context/offer-angles.md` exists).
Also used by offer-maker E02 as a validation gate after angle extraction.

## Purpose

Validates that extracted offer angles are specific, provable, and ready for headline/description composition.

## Checklist (30 items — 5 per angle)

### Problem/Pain Points
- [ ] Specific frustrations identified (not generic)
- [ ] Language matches how customers describe the problem
- [ ] Failed alternatives documented
- [ ] Cost of inaction articulated
- [ ] 2-3 headline-ready phrases extracted (each <=30 chars)

### Value Proposition
- [ ] Main benefit is clear (not just features)
- [ ] Understandable in 5 seconds
- [ ] Specific to target audience
- [ ] Differentiated from competitors
- [ ] 2-3 headline-ready phrases extracted (each <=30 chars)

### USPs (Unique Selling Points)
- [ ] Actually unique (competitors cannot claim it)
- [ ] Provable (not just asserted)
- [ ] Matters to target customer
- [ ] Quantified where possible
- [ ] 2-3 headline-ready phrases extracted (each <=30 chars)

### Value Boosters
- [ ] Everything included is listed
- [ ] Quantities/specifics documented
- [ ] Would surprise customer positively
- [ ] Each element is named and specific
- [ ] 2-3 headline-ready phrases extracted (each <=30 chars)

### Social Proof & Authority
- [ ] Numbers documented (not "trusted by many")
- [ ] Ratings and review counts included
- [ ] Recognizable names/logos documented (if B2B)
- [ ] Credentials that matter documented
- [ ] 2-3 headline-ready phrases extracted (each <=30 chars)

### Risk Removal
- [ ] Guarantee/trial is specific (terms, duration)
- [ ] Addresses main hesitation
- [ ] Stronger than competitors (or at parity)
- [ ] Terms are clear and frictionless
- [ ] 2-3 headline-ready phrases extracted (each <=30 chars)

## Gap Identification

| Missing/Weak Angle | What It Means | Action |
|-------------------|---------------|--------|
| No clear pain points | Don't understand customer | Talk to customers, read reviews |
| No clear USP | Commoditized offer | Return to offer design (`/offer-maker create`) |
| No value boosters | Offer is thin | Consider what could be added/bundled |
| No social proof | Haven't collected it | Start collecting reviews immediately |
| No authority | Haven't built credentials | Pursue certifications, awards |
| No risk removal | Asking too much commitment | Add guarantee, trial, or easier first step |

## Definition of Done

Angles are ready for RSA composition (`/rsa-maker`) when:
- [ ] All 6 angles documented with proof points
- [ ] Each angle has 2-3 headline-ready phrases (<=30 chars each)
- [ ] Gaps identified and either fixed or flagged
- [ ] Slot distribution mapped (7-8 total headlines)
