# Geo-Schedule Auditor — Demographics & Advanced Diagnostics (GS-D10–D14)

**Threshold source:** Diagnostic Thresholds Reference (REF_69), Demographics section + cross-references

---

## GS-D10: Demographic CPA Outliers
**Severity:** Medium (5 pts) | **Data:** `demographics-age.csv`, `demographics-gender.csv`, `demographics-income.csv`

**Rule:** Demographic segments (age, gender, income) with CPA 50%+ above account average are underperforming and should be reviewed for bid adjustment or exclusion.

**How to check:**
1. Read the three demographics CSVs → aggregate by each demographic dimension separately:
   - Age ranges: `demographics-age.csv` → group by `ad_group_criterion.age_range.type`
   - Gender: `demographics-gender.csv` → group by `ad_group_criterion.gender.type`
   - Income: `demographics-income.csv` → group by `ad_group_criterion.income_range.type`
2. Calculate account-level average CPA
3. For each segment with 100+ clicks, compute deviation: `(segment_CPA - avg_CPA) / avg_CPA * 100`
4. WARN if any segment CPA is 50%+ above average

**Pass/Fail:**
- PASS: All demographic segments within 50% CPA variance
- WARN: Segments with CPA 50%+ above average — list each with dimension, segment, CPA, deviation, clicks
- SKIP: All segments have <100 clicks → "Insufficient demographic data"

**Details format:** "[Dimension] [Segment]: CPA $X (XX% above avg $Y) on Z clicks"

**Routing on WARN:** `/geo-schedule-optimizer demo` to apply demographic bid adjustments

## GS-D11: Smart Bidding Modifier Conflict
**Severity:** Low (3 pts) | **Data:** `campaign-criteria.csv` + `campaigns.csv`

**Rule:** Non-zero bid modifiers (except -100%) on Smart Bidding campaigns are silently ignored by the algorithm. They create false confidence — the modifier appears set but has no effect. From BID-D18 in Diagnostic Thresholds Reference.

**How to check:**
1. Read `campaigns.csv` → identify campaigns using Smart Bidding strategies:
   - `TARGET_CPA`, `TARGET_ROAS`, `MAXIMIZE_CONVERSIONS`, `MAXIMIZE_CONVERSION_VALUE`
2. Read `campaign-criteria.csv` → for those campaigns, find all criteria with `bid_modifier != 0` AND `bid_modifier != 1.0` (no modifier) AND `bid_modifier != 0.0` (-100%, which IS respected for device/demographic criteria only — NOT for ad_schedule or location)
3. WARN if any non-zero, non-100% modifiers exist on Smart Bidding campaigns

**Pass/Fail:**
- PASS: No Smart Bidding campaigns have non-exclusion modifiers, OR no Smart Bidding campaigns exist
- WARN: Smart Bidding campaigns with ignored modifiers — list campaigns and modifier details
- SKIP: Never (always runs)

**Details format:** "[Campaign] (TARGET_CPA): has X active modifiers that are ignored by Smart Bidding. Only -100% (exclusion) modifiers take effect."

**Routing on WARN:** `/geo-schedule-optimizer` to remove ignored modifiers from Smart Bidding campaigns

## GS-D12: Seasonal Geographic Patterns
**Severity:** Medium (5 pts) | **Data:** `geo-seasonal-comparison.csv` (from `analyze-geo-seasonal.js`)

**Rule:** If a location's CPA deviates more than 30% from the same period last year, it may indicate a seasonal shift that requires proactive bid adjustment.

**How to check:**
1. Read `geo-seasonal-comparison.csv` → generated by `analyze-geo-seasonal.js`
2. If the file contains `# insufficient_data` → SKIP
3. For each location with `verdict = WARN`:
   - Check `yoy_deviation_pct` — positive means CPA increased (worse), negative means decreased (better)
   - Flag locations with >30% YoY CPA increase as seasonal risk
   - Flag locations with >30% YoY CPA decrease as seasonal opportunity
4. Classify: worsening locations = risk, improving locations = opportunity

**Pass/Fail:**
- PASS: No locations with >30% YoY CPA deviation
- WARN: Locations with significant seasonal shifts — list each with current CPA, prior CPA, deviation
- SKIP: <12 months of data → "Insufficient historical data for YoY comparison"

**Details format:** "[Location]: CPA $X now vs $Y last year (±Z% YoY). Seasonal [risk/opportunity]."

**Routing on WARN:** "Flag for seasonal review — no automated action recommended"

## GS-D13: Demographic Exclusion Opportunity
**Severity:** High (10 pts) | **Data:** `demographics-age.csv`, `demographics-gender.csv`, `demographics-income.csv`

**Rule:** Demographic segments with CPA 50%+ above average, sustained over 60+ days, with 100+ clicks represent strong exclusion candidates. Unlike GS-D10 (which is a point-in-time warning), this check requires sustained poor performance.

**How to check:**
1. Read the three demographics CSVs → aggregate by demographic dimension
2. Filter to segments with 100+ clicks
3. Check the data period: must span 60+ days to qualify
   - If data period < 60 days → SKIP for this check (GS-D10 still applies as a warning)
4. For segments with CPA 50%+ above average sustained across the full period:
   - These are strong exclusion candidates via -100% bid modifier
5. FAIL if any segment meets all criteria (50%+ CPA, 60+ days, 100+ clicks)

**Pass/Fail:**
- PASS: No demographic segments meet the sustained poor performance criteria
- FAIL: Segments with sustained CPA 50%+ above average — strong exclusion candidates
- SKIP: Data period <60 days → "Need 60+ days to confirm sustained poor performance"

**Details format:** "[Dimension] [Segment]: CPA $X (XX% above avg) sustained over Y days on Z clicks. Exclusion candidate."

**Routing on FAIL:** `/geo-schedule-optimizer demo` to apply demographic exclusions (-100%)

## GS-D14: Geographic Targeting Optimization Opportunity
**Severity:** Medium (5 pts) | **Data:** `geo-user-location.csv` + `campaigns.csv`

**Rule:** High-performing locations (CPA 20%+ below average) that are also impression-share limited (IS lost to budget >10%) are candidates for dedicated campaigns or budget expansion. This extends GS-D04 with specific actionable guidance.

**Note:** No explicit threshold in Diagnostic Thresholds Reference. Derived from GS-D04 pattern.

**How to check:**
1. Same data and logic as GS-D04
2. **IS check:** The Google Ads API does NOT support impression share on `geographic_view` or `location_view`. Use campaign-level IS from `campaigns.csv` as a proxy.
3. For locations that qualify (CPA 20%+ below avg in campaigns with IS lost to budget >10%):
   - Estimate opportunity from campaign-level IS data
   - `additional_conv_estimate = current_conversions * (campaign_IS_lost_pct / campaign_IS_pct)`
4. WARN with advisory guidance (this is an opportunity, not a problem)

**Pass/Fail:**
- PASS: No high-performing locations are in IS-limited campaigns, OR GS-D04 already captured this
- WARN: Opportunity to scale — list locations with estimated additional conversions
- SKIP: No campaign-level IS data → "IS data not available"

**Details format:** "[Location]: CPA $X (Y% below avg), IS lost to budget Z%. Estimated additional conversions: ~N/month with dedicated campaign."

**Routing on WARN:** "Advisory: consider dedicated campaign for high-performing locations"
