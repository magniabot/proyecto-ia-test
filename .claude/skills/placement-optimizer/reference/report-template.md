# Placement Optimizer — Report Template

## Changelog: `context/analysis/placement-changelog.md`

Each optimizer run appends an entry (auto-generated by `mutate.js` in live mode):

```markdown
## {YYYY-MM-DD} — Placement Optimizer

**Mode:** live | **Account:** {customer_id}
**Result:** {applied}/{total} applied
**Source:** context/analysis/placement-audit.md

| # | Entity | Action | Category | Target | Status |
|---|--------|--------|----------|--------|--------|
| 1 | customer_negative_criterion | create | mobile_app_exclusion | Games - Action & Adventure | OK |
| 2 | shared_set | create | exclusion_list_create | PPCOS — Performance Exclusions | OK |
| 3 | shared_criterion | create | placement_exclusion | spam-domain.xyz | OK |
| 4 | campaign_shared_set | create | list_linkage | Link 'Performance Exclusions' to 'Display' | OK |
| 5 | campaign | update | brand_safety_setting | Video campaign inventory type | OK |
```

## Operations File: `created/placement-ops/operations-{YYYY-MM-DD}.json`

Preserved for audit trail. Contains all planned operations with metadata.

## Memory Log

After successful live apply, log to `context/memory/YYYY-MM-DD.md`:

```markdown
## Placement Optimizer Changes
- Applied {n} operations to account {customer_id}
- {n} mobile app categories excluded (PL-E01)
- {n} placements added to exclusion list (PL-E02/E03/E04)
- {n} campaign-list links created (PL-E06)
- Brand safety: {n} campaigns updated to Standard inventory (PL-E05)
- Operations file: created/placement-ops/operations-{date}.json
```
